# Project Rules

This file defines what `melissa-portfolio-site` owns and what should stay elsewhere.

## This Project Owns

- the public website code
- portfolio page structure
- public copy formatting and adaptation
- imagery integration
- accessibility QA and remediation
- responsive layout improvements
- launch and deployment readiness

## This Project Does Not Own

- private case-study source material
- interview-only stories and evidence
- resume tailoring
- application strategy
- deep brand-system exploration

## Boundary Rules

- if the work is about rewriting the underlying story, move to `2026-job-search/`
- if the work is about refining tokens, brand direction, or identity rules, move to `melissa-about/`
- if the work is about publishing, structuring, styling, or QA-ing the public version, keep it here

## Good Thread Examples

- add a new Selected Work page from an approved case study
- improve mobile layout on the About page
- add imagery, captions, and alt text across the site
- run a portfolio accessibility cleanup pass

## Not-Great Thread Examples For This Project

- invent a new case-study strategy from raw notes
- decide overall career positioning from scratch
- tailor a resume for a role
- explore broad brand directions with no site implementation goal
