# Selected Work Tag System

This reference defines the public portfolio tag system for selected work cards and case-study hero metadata.

## Purpose

Tags should help visitors scan the kind of work each project represents. Use them for public-facing categories, not internal project mechanics.

## Rules

- Use 2-5 tags per selected work page.
- Keep labels short, public-facing, and title case.
- Treat the individual selected work page as the source of truth for each project's tags.
- Keep the matching card on `selected-work.html` in the same tag order as the individual page.
- Use the canonical labels below before creating a new one.
- Add a new tag only when it describes a repeatable portfolio category.
- Add new tag colors through `styles/site.css` token pairs before applying a new class.

| `tag-ux` | Sage | UX design, UX leadership, interaction design, and research-informed experience work |

## When To Add A New Tag

Create a new tag when a new category will likely appear on more than one selected work page. For one-off context, use the project copy instead of adding another visual category.
