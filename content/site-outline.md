## Portfolio Design System Governance

This portfolio uses the design-system adapter framework as a governance layer.

- Carbon influence: enterprise spacing discipline, action hierarchy, component structure, metadata patterns, and reusable layout primitives.
- USWDS influence: visible focus, plain language, persistent form labels, readable help text, practical validation behavior, and inclusive content structure.
- Adapter framework influence: every major UI pattern should connect to tokens, components, accessibility guidance, and reusable pattern documentation.

Core reusable patterns:

- Tokens: CSS custom properties for color, typography, spacing, radius, focus, and layout width.
- Navigation: semantic header and `nav`, descriptive links, current page state, visible keyboard focus.
- Buttons: native buttons for form actions, links for navigation, primary and secondary visual hierarchy.
- Cards and panels: repeated containers for work summaries, status callouts, and metadata.
- Forms: visible labels, contextual hints with `aria-describedby`, required fields, and CSS support for `aria-invalid`.
