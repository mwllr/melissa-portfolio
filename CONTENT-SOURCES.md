# Content Sources

This file explains where portfolio-site content should come from.

## Main Source Projects

### `2026-job-search/`

Primary source for:

- case-study source files
- interview-story material
- recruiter-friendly narratives
- role-aligned positioning language

Use this project when the content needs to be expanded, rethought, or rewritten at the source level.

### `melissa-about/`

Primary source for:

- brand direction
- token decisions
- voice and identity foundations
- visual language guidance

Use this project when the work changes the underlying brand system rather than just a page implementation.

## Site Project Role

`melissa-portfolio-site/` is the public presentation layer.

`content/site-outline.md` records the site-side positioning, voice, proof-sequence, and public-evidence guidance used to adapt that material. It does not replace the source-of-truth ownership described above.

Use it to:

- adapt source material for concise public pages
- simplify and structure content for web reading
- apply approved brand and token choices
- integrate imagery and supporting visuals
- ensure accessibility and responsiveness

## Practical Flow

1. Develop or rewrite source material in `2026-job-search/` when needed.
2. Confirm any brand or token changes in `melissa-about/` when relevant.
3. Adapt the public-safe version in `melissa-portfolio-site/`.
4. QA the result for clarity, accessibility, and visual consistency.
