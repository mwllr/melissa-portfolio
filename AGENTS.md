# Melissa Portfolio Site Project Instructions

## Project Purpose

This project owns the public-facing portfolio site.

Use it for:

- site implementation
- page layout and styling
- accessibility improvements
- imagery integration
- content adaptation for public presentation
- launch-readiness work

Do not use this project as the main place for deep source-content development.

## Source-Of-Truth Boundaries

- `2026-job-search/` owns case-study source material, private notes, interview stories, and resume-related content
- `melissa-about/` owns brand foundations, token direction, and identity rules
- `melissa-portfolio-site/` owns the public implementation and presentation layer

When content needs major rewriting or strategic reframing, do that in the source project first and then adapt it here.

## Default Behavior

When working in this project:

1. preserve the site's public-facing purpose
2. prefer adapting approved source content over inventing entirely new narratives here
3. improve structure, readability, accessibility, and presentation
4. keep private or overly detailed material out of public pages
5. use the task list as the implementation backlog

## Content Rules

- public portfolio pages can be shortened, clarified, and formatted for the web
- sensitive, internal, or interview-only details should stay outside the public site
- major case-study rewrites should happen in `2026-job-search/`
- major brand or token decisions should happen in `melissa-about/`

## Working Style

- keep implementation simple and maintainable
- preserve semantic HTML and accessible structure
- prefer clear, polished public presentation over exhaustive detail
